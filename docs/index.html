开发流程

按这条路径走，可以先把最小闭环跑通，再进入代码和流程定义文件。每一步都对应仓库里的真实配置，不需要额外脚手架。

01

准备环境

本地开发使用 Java 8、Maven 和 MySQL。数据库名是 javapub-flowable2，Flowable 表会在首次启动时自动创建。

02

配置连接

通过 MYSQL_URL、MYSQL_USERNAME、MYSQL_PASSWORD 覆盖默认配置，避免把本机密码提交到仓库。

03

启动服务

调试代码用 mvn spring-boot:run；想快速拉起完整环境，用 docker compose up --build。

04

验证接口

先发起报销流程，再查询发起人的待办，最后完成任务。金额小于等于 500 走经理审批，超过 500 走老板审批。

05

查看流程图

通过 /expense/processDiagram 查看当前流程位置。流程结束后接口可能返回空内容，这是代码里的预期行为。

06

记录变更

涉及功能、依赖、配置或行为变更时，在 docs/history/ 下新增历史记录，保留改动背景和验证结果。

启动方式

项目保留本地运行和 Docker Compose 两种路径。前者适合开发调试，后者适合演示、交接和新机器快速启动。

本地运行

确认 MySQL 已创建数据库，然后用 Maven 启动应用。

CREATE DATABASE `javapub-flowable2`
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_general_ci;

MYSQL_USERNAME=root MYSQL_PASSWORD=你的密码 \
mvn spring-boot:run

Docker Compose

同时启动应用和 MySQL，默认暴露应用端口 8081。

docker compose up -d --build
docker compose logs -f app
docker compose down

接口验证

启动成功后访问 http://localhost:8081。为了方便快速体验，示例接口使用 GET 请求。

发布 Pages

这个页面就是 GitHub Pages 的入口文件。仓库设置完成后，会从 docs/index.html 直接发布。

GitHub 设置

  • 进入仓库的 Settings。
  • 打开 Pages。
  • Source 选择 Deploy from a branch。
  • Branch 选择 main，目录选择 /docs。

发布地址

部署完成后访问：

https://rodert.github.io/springboot-flowable/

如果页面没有立即更新，通常是 GitHub Pages 构建和 CDN 缓存还没刷新，等一两分钟再看。
