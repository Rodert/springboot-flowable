Flowable Starter Guide

springboot-flowable 开发流程

这是一个 Spring Boot 整合 Flowable 的入门示例。页面把从拉代码、配置数据库、启动服务、验证接口到提交记录的流程串起来，方便新同学快速跑通。

查看 GitHub 仓库 阅读 README JSON 格式化工具

推荐开发节奏

01

准备基础环境

本地开发需要 JDK 8、Maven 和 MySQL。数据库名使用 javapub-flowable2，Flowable 表会在首次启动时自动创建。

CREATE DATABASE `javapub-flowable2`
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_general_ci;
02

配置连接信息

优先通过环境变量覆盖数据库连接，避免把本机密码写进仓库。

MYSQL_URL=jdbc:mysql://localhost:3306/javapub-flowable2
MYSQL_USERNAME=root
MYSQL_PASSWORD=你的密码
03

启动应用

本地启动适合调试代码，Docker Compose 适合快速拉起完整环境。

mvn spring-boot:run

# 或者
docker compose up --build
04

验证报销流程

先发起流程，再查询待办，随后完成“出差报销”任务，流程会按金额进入经理或老板审批。

GET /expense/add?userId=123&money=2000&descption=出差打车
GET /expense/list?userId=123
GET /expense/apply?taskId=任务ID
05

记录变更

涉及功能、依赖、配置或行为变更时，在 docs/history/ 下新增一份历史记录，方便后续追踪上下文。

docs/history/YYYY-MM-DD-change-summary.md

项目速览

核心流程是“出差报销审批”：金额不超过 500 走经理审批，超过 500 走老板审批，驳回后回到发起人任务。

  • Java 8
  • Spring Boot 2.7.18
  • Flowable 6.8.1
  • MySQL 8.0
  • Maven
  • Docker Compose

发布到 GitHub Pages

  • 进入 GitHub 仓库的 Settings。
  • 打开 Pages 配置。
  • Source 选择 Deploy from a branch。
  • Branch 选择 main，目录选择 /docs。
  • 保存后访问 https://rodert.github.io/springboot-flowable/。

常用命令

# 打包
mvn clean package -DskipTests

# 执行测试
mvn test

# Docker 启动
docker compose up -d --build

# 查看应用日志
docker compose logs -f app

# 停止服务
docker compose down
